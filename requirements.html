Mapping Design Decisions to Functional and Non-Functional Requirements

This section elucidates the direct correlation between the architectural and technological choices made in the design of our video streaming platform and the functional and non-functional requirements that drive its development. This mapping ensures that the platform's architecture effectively addresses the core functionalities and quality attributes essential for its success.

Functional Requirements Alignment
Video Streaming (FR-1):
  • The platform's support for real-time live streaming is facilitated by the selection of a real-time streaming protocol, such as WebRTC, and potentially the integration of a message queue like Kafka. On-demand playback is enabled by the Video Management Service, the Content Delivery Network (CDN), and streaming protocols like HLS and DASH. Low-latency delivery is achieved through CDN utilization, optimized video encoding via the Media Processing Service, and the efficiency of the chosen streaming protocols.
Social Media Integration (FR-2):
  • Video sharing capabilities are provided through backend APIs and frontend implementations that facilitate seamless integration with external social media platforms. The User Management Service handles the logic for user following and activity updates. User preferences and interactions, such as video saving and liking, are stored and managed within the backend database.
Content Recommendation (FR-3):
  • Personalized video suggestions are powered by the Recommendation Engine, which may leverage machine learning models to analyze user data and provide tailored recommendations. The integration of machine learning algorithms within the Recommendation Engine enhances the accuracy and relevance of content delivery.
Interactive Comments (FR-4):
  • Real-time bullet comments are implemented using WebSockets or Server-Sent Events (SSEs) to provide an interactive viewing experience. Timestamped comments are supported by storing comment data with timestamps in the backend, allowing for contextual retrieval and display.
Video Playback and Controls (FR-5):
  • Standard video playback controls, including play, pause, seek, volume adjustment, and full-screen mode, are implemented within the frontend using React.js.
Subtitle Translation (FR-6):
  • Automatic subtitle translation is facilitated by integrating with a cloud-based translation service, such as the Google Cloud Translation API, within the Media Processing Service.
User Activity and Interaction (FR-7):
  • The backend tracks user activity and provides this data to the frontend for display, potentially using WebSockets for real-time updates, enhancing the sense of community and engagement.
Payment and Subscription (FR-8):
  • The Monetization Service manages subscription tiers and payment processing, potentially integrating with payment gateways like Stripe or PayPal to provide secure and convenient payment options.
Moderation and Compliance (FR-9):
  • Moderation tools and policies are implemented to ensure platform integrity and user safety. This may involve human moderators or automated systems utilizing machine learning for content analysis.
Shorts (FR-10):
  • The Video Management and Media Processing services handle the specific requirements for processing and delivering short-form video content.
Affiliate Links for Content Creators (FR-11):
  • The backend stores and manages affiliate link data, while the frontend ensures the appropriate display and functionality of these links within creator content.
Non-Functional Requirements Alignment
Scalability (NFR-1):
  • The microservices architecture, CDN utilization, and cloud infrastructure (AWS) contribute to the platform's ability to accommodate a growing user base and content volume without compromising performance. Scalable databases like Amazon RDS and Redis further enhance this capability.
Performance (NFR-2):
  • React's efficient rendering, CDN usage, optimized video encoding, and database caching with Redis contribute to rapid load times and responsive interactions, ensuring a positive user experience.
Security (NFR-3):
  • The comprehensive Auth Service, secure coding practices, and data protection measures address security concerns by safeguarding user data, preventing unauthorized access, and mitigating cyber threats.
Usability (NFR-4):
  • The choice of React.js for frontend development, coupled with a focus on UI/UX design principles, ensures an intuitive and user-friendly interface that is accessible to users with diverse technical backgrounds.
Reliability (NFR-5):
  • Leveraging cloud infrastructure (AWS) with its high-availability services, along with robust monitoring and system redundancy, contributes to maintaining high availability and minimizing downtime.

Supporting Services: Enabling Essential Functionalities

Auth Service:

This service forms the foundation of platform security, ensuring secure authentication and authorization of users. It enables Single Sign-On (SSO), allowing users to leverage their existing accounts from providers like Google or Facebook for streamlined access. Security Groups are employed to control network access to the authentication infrastructure, while Identity and Access Management (IAM) provides granular control over user roles and permissions. To further enhance security, a VPN may be utilized to secure remote access for administrators.

Analytics Service:

This service functions as the platform's data intelligence hub, collecting and analyzing user data and platform usage patterns. By providing insights into popular content, user engagement metrics, and overall platform performance, it informs decision-making regarding content strategy, platform enhancements, and the improvement of recommendation algorithms.

Monetization Service:

Should the platform employ a monetization strategy, this service manages the associated processes. It handles subscription plans, payment processing, and potentially the serving of advertisements. Integration with payment gateways like Stripe or PayPal ensures secure and convenient payment options for users.
